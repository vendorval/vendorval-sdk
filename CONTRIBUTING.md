# Contributing

Thanks for your interest in improving the VendorVal SDKs! The Node and Python clients both live in this repository and ship the same API surface.

## Getting started

**Node (`packages/node`):**

```bash
pnpm install
pnpm -r build
pnpm -r test
pnpm --filter vendorval-sdk typecheck
```

**Python (`packages/python`):**

```bash
cd packages/python
uv sync
uv run pytest
uv run ruff check src tests
uv run mypy src
```

## Guidelines

- **Keep the two SDKs at parity.** If you add or rename a public type or method on one side, mirror it on the other. The `type-parity` check (`node scripts/check-type-parity.mjs`) runs in CI.
- **Lint and types must pass.** Python uses `ruff` and `mypy --strict`; TypeScript uses `tsc --noEmit`. CI runs the full matrix (Node 18/20/22, Python 3.9–3.13).
- **Don't hand-edit `specs/openapi.json`** — it is generated by `node scripts/sync-openapi.mjs`.
- **Changelogs:** add a note to the relevant `packages/*/CHANGELOG.md`. Write entries for SDK consumers — describe the user-facing change, not internal implementation details.
- **Commits & PRs:** use conventional commit messages and open a pull request against `main`. CI must be green before merge.

## Reporting issues

Please file bugs and feature requests on the [issue tracker](https://github.com/Modali-Consulting/vendorval-sdk/issues).
